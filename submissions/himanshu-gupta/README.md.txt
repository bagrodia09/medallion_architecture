Brief description of the approach:
The code implements an end‑to‑end Medallion Architecture (Bronze → Silver → Gold) data pipeline using PySpark and Delta Lake. Raw order data is first ingested into a Bronze Delta table exactly as received, with added ingestion metadata for traceability. The Silver layer then reads from Bronze to enforce data quality by deduplicating records, applying proper data types, and separating valid records from invalid ones into a quarantine table with explicit reasons. Incremental changes are handled in Silver using Delta MERGE to support updates and inserts. Finally, the Gold layer derives business‑ready aggregates for analytics, such as daily revenue by region and total customer spend. Assertions at the end verify record counts across all layers to ensure correctness and pipeline integrity.
